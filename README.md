# AI 旅行规划师 — Hermes Agent Skill

基于 Hermes Agent 框架的全生命周期智能旅行助手。从出发前规划、旅途中实时调整，到旅行后生成手账回忆，覆盖一趟旅行的完整生命周期。

## 核心功能

### 出发前：引导式行程规划
- **7步信息采集**：城市、出发日期、出行方式、天数、预算、同行人、偏好
- **天气感知**：15天内出发自动查询真实天气；极端天气主动建议改期；15天外生成晴雨双方案
- **交通查询**：高铁/飞机支持班次和票价搜索
- **联网搜索**：实时搜索公开旅游攻略 + 高赞抖音/小红书视频链接
- **476个中国城市**全覆盖，200个景点别名映射

### 旅途中：实时动态调整（迭代1）
7种触发词，一句话调整后续行程：累了/下雨了/想吃X/不想去X了/还有时间/太贵了/改一下明天

### 长期记忆（迭代2）
- 每次旅行后自动学习偏好，下次"老样子"跳过重复输入
- 6维度用户画像

### 旅行后：手账生成（迭代3）
- 行程最后一天主动提醒
- 照片配背后故事，不用星级评分
- 输出 Markdown 手账，保存到 journals/ 目录

## 飞书适配说明

此 Skill 在设计之初便规划了飞书/Lark 端的全流程适配——精简提问、卡片输出、一步到位。桌面端早已验证通过，运行丝滑。

然而，在飞书网关的实际测试中，由于平台上存在研发更早、沉淀更深的其他 Skill，本 Skill 未能成功夺过对话的主导权。Agent 在接收到旅行意图时，时常被更具先发优势的 Skill 抢先响应，导致精心设计的引导式采集流程无法完整呈现。

**这不是失败，这是登顶前的最后一次休整。**

飞书的代码接口我已摸透，卡片模板就绪，网关路径清晰——缺的只是时间沉淀和一次平台级的优先级重构。这一仗暂挂免战牌，但剑已磨好。

> **待我重整旗鼓，飞书端必是第一站。届时，任何 Skill 都无法从我手中抢走一句话。**

—— 2026年夏，于飞书城下
